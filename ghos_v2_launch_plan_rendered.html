Phased Rollout Strategy

GHOS V2 is a complete clinic management system with queue management, patient PWA, reporting, inventory, and analytics.

We launch with 1–2 core departments (Echo/ECG) in full production, while all other departments display as Coming Soon — Under Construction. This enables live testing at small scale — 10–12 patients/day — with real-time feedback for rapid iteration.

As each department stabilizes, we activate the next — one at a time — ensuring quality at every step.

  • 1
    Phase 1 — Week 1
    Deploy Echo/ECG department live. Register 10+ patients daily. Monitor all transitions.
  • 2
    Phase 2 — Week 2+
    Activate General Medicine. Expand patient volume gradually.
  • 3
    Phase 3 — Month 2
    Add Orthopedics, Pediatrics, Ophthalmology departments.
  • 4
    Phase 4 — Month 3+
    Full hospital-wide rollout with all departments and modules live.

Key Features Ready at Launch

  • Queue Management — Real-time token system with call/start/complete/deliver workflow
  • Patient PWA — Status tracking, token slip, reports view, timeline, feedback
  • Multi-role Dashboards — Reception, Technician, Doctor, Manager views
  • Reports & Analytics — Dashboard with live KPIs and trends
  • Patient Registration — Visit management, bulk CSV import
  • Notes & Report Workflow — Clinical notes with full status lifecycle

Live Testing Protocol

| Step | Action | |------|--------| | 1 | Start with 10–12 patients/day in Echo/ECG only | | 2 | Monitor all 6 status transitions: WAITING → DELIVERED | | 3 | Track errors, bottlenecks, UX issues daily | | 4 | Fix & deploy before adding next department |

Risk Mitigation

  • Phased rollout limits blast radius — issues stay contained to active departments only
  • Under Construction banners prevent confusion for unavailable features
  • Small patient volume (10–12/day) allows controlled, safe testing
  • Daily monitoring with immediate hotfix capability
